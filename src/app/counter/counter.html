NGRX: follows the Redux pattern.

  • Store: central container for application state
  • Actions: describing what happend. Triggered by user Event
  • Reducers: pure functions to transit to a new state. Specify how the state changes in response to the Actions
  • Selectors (Optional): pure functions to read part of the state
  • Effects (Optional): handing side effects (Http requests, local storage...)

User event => action dispatched => trigger reducers => update state in store
